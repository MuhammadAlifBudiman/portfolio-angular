# Design Rule

- Review `DESIGN.md` before visual edits.
- Keep colors attached to existing Tailwind/SCSS tokens.
- Do not add generic gradients, decorative blobs, AI-sounding copy, or fake social proof.
- Maintain dark and light theme readability.
- Use semantic HTML for interactive UI whenever feasible.
- Any UI change must describe its mobile behavior.
