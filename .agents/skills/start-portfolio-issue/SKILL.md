# Start Portfolio Issue

Use this workflow before implementation unless the user explicitly authorizes skipping issue creation.

1. Restate the requested outcome and affected routes/components.
2. Map the work to `docs/product/FEATURE_SPEC.md`.
3. If no Feature ID fits, propose the smallest spec update first.
4. Check `docs/product/ACCEPTANCE_CRITERIA.md`, `docs/ROADMAP.md`, and `docs/TRACEABILITY_MATRIX.md`.
5. Identify or draft a GitHub Issue using `docs/templates/portfolio-feature-issue.md`.
6. Link the issue to `Portfolio V1.0` when available.
7. Return Feature ID, issue number or draft, files likely to change, verification commands, risks, and out-of-scope items.
